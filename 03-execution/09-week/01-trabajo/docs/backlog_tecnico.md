# Backlog técnico para estabilización de la base de datos

## Historias de usuario

### HU-001: Identificar dominios
Como desarrollador, quiero identificar los dominios funcionales del modelo para comprender su estructura.

### HU-002: Estructura del repositorio
Como desarrollador, quiero organizar la estructura del repositorio con ramas develop, qa y main para gestionar el versionamiento.

### HU-003: Contenerización de PostgreSQL
Como desarrollador, quiero levantar la base de datos en un contenedor para facilitar su despliegue local.

### HU-004: Integración de Liquibase
Como desarrollador, quiero integrar Liquibase para versionar el DDL de la base de datos.

### HU-005: Organización de changelogs
Como desarrollador, quiero separar el DDL en changelogs por dominio para mantener el orden del proyecto.

### HU-006: Roles y permisos
Como desarrollador, quiero implementar roles y permisos diferenciados para mejorar la seguridad.

### HU-007: Datos de prueba
Como desarrollador, quiero generar datos de prueba organizados para validar el sistema.

### HU-008: Documentación técnica
Como desarrollador, quiero documentar las decisiones y avances para mantener trazabilidad.
