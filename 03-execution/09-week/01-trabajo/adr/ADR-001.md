# ADR-001: Ampliación de un nuevo dominio funcional

## Contexto
El sistema actual presenta múltiples dominios funcionales relacionados con la gestión de aerolíneas, clientes, vuelos, reservas, pagos y facturación. Sin embargo, no se evidencia un dominio enfocado en la gestión de notificaciones a los usuarios.

## Problema
El sistema no cuenta con un mecanismo estructurado para el envío y gestión de notificaciones a los usuarios, lo cual limita la comunicación de eventos importantes como confirmaciones de reserva, cambios de vuelo o estados de pago.

## Decisión
Se propone la creación de un nuevo dominio funcional denominado "Notificaciones", el cual permitirá gestionar el envío de mensajes a los usuarios a través de diferentes canales.

## Justificación técnica
La implementación de este dominio permitirá desacoplar la lógica de notificaciones del resto del sistema, facilitando su mantenimiento y escalabilidad. Además, permitirá centralizar la gestión de comunicaciones con los usuarios.

## Consecuencias
Se incrementa la complejidad del sistema al agregar un nuevo dominio, pero se mejora la organización, mantenibilidad y capacidad de evolución del proyecto.
