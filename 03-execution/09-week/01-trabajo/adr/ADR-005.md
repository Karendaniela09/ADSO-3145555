# ADR-005: Contenerización de la base de datos con Docker

## Contexto
El proyecto requiere una forma estandarizada de levantar la base de datos en diferentes entornos de desarrollo y pruebas.

## Problema
No existe una estrategia definida para la configuración y despliegue de la base de datos, lo que puede generar inconsistencias entre entornos.

## Decisión
Se propone utilizar contenedores Docker para desplegar PostgreSQL, garantizando un entorno controlado y reproducible.

## Justificación técnica
El uso de Docker permite estandarizar el entorno de ejecución, facilitando la configuración, despliegue y portabilidad del sistema.

## Consecuencias
Se mejora la consistencia entre entornos y la facilidad de despliegue, aunque se requiere conocimiento básico en el uso de contenedores.
