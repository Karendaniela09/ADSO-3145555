# ADR-002: Manejo de roles con permisos diferenciados

## Contexto
El sistema cuenta con estructuras básicas de seguridad como roles, permisos y usuarios, pero es necesario fortalecer el control de acceso para mejorar la seguridad y trazabilidad.

## Problema
No se ha definido una estrategia clara para asignar permisos diferenciados a los roles, lo que puede generar accesos indebidos o falta de control en las operaciones del sistema.

## Decisión
Se establece una estrategia de control de acceso basada en roles (RBAC), donde los permisos serán asignados a los roles y estos a su vez a los usuarios.

## Justificación técnica
El uso de RBAC permite una administración más organizada y escalable de los permisos, facilitando el control de acceso y la trazabilidad de las acciones realizadas por los usuarios.

## Consecuencias
Se mejora la seguridad del sistema y la trazabilidad de las acciones, aunque requiere una adecuada gestión de roles y permisos para evitar configuraciones incorrectas.LISTO
