# ADR-003: Implementación de Liquibase para versionamiento del DDL

## Contexto
El modelo de base de datos actual está definido en un único script SQL, lo que dificulta el control de cambios y el versionamiento del esquema.

## Problema
No existe un mecanismo de control de versiones para el DDL, lo que puede generar inconsistencias entre entornos y dificultar la evolución del modelo de datos.

## Decisión
Se implementará Liquibase como herramienta para gestionar el versionamiento del DDL mediante el uso de changelogs organizados por dominios funcionales.

## Justificación técnica
Liquibase permite llevar un control detallado de los cambios en la base de datos, facilitando la trazabilidad, despliegue y mantenimiento del sistema.

## Consecuencias
Se mejora el control de versiones y la consistencia entre entornos, aunque implica una curva de aprendizaje y la necesidad de mantener organizados los changelogs.
